4. 名前に意味を

  • ファイルやdirectoryに意味のわかる名前をつけよう．
  • そうすると検索の手間が減るから．
  • tree -dとかが役に立ちます

5. 自分に合わせてdirectory構成を

  • 授業で提供している順番にそれほど意味はありません
  • 自分の知識の構造に合わせてfile構造を作ってください．
  • 例えば，apt, emacs, fish, rubyなんかの項目に分けてください．
